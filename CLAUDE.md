## Stack

- **Next.js 16** with App Router and Turbopack
- **React 19**
- **Tailwind CSS v4** (imported via `@import "tailwindcss"` in `globals.css`, no config file needed)
- **shadcn/ui** for UI components (see below)
- **TypeScript** — strict mode. Path alias `@/*` maps to the **repo root** (`tsconfig.json` has `"@/*": ["./*"]`), not `./src/*`. This is the `create-next-app` default alias left uncustomized while `src/` was enabled, so the two don't line up: code lives in `src/`, but `@/` resolves above it. Every alias therefore needs the `src/` prefix — e.g. `@/src/lib/utils`, `@/src/components/ui`.

## UI Components (shadcn/ui)

Build UI from **shadcn/ui** components — do not hand-roll structural or interactive elements (buttons, inputs, sidebars, dialogs, dropdowns, etc.) with raw HTML + Tailwind when a shadcn primitive exists.

- Add components with `pnpm dlx shadcn@latest add <component>` — do not write them by hand.
- Config lives in `components.json`. Because `@/*` maps to the repo root, aliases are prefixed with `src/` (e.g. `@/src/components/ui`, `@/src/lib/utils`).
- Generated primitives live in `src/components/ui/` and are vendored verbatim — do not edit them; they are re-generated by the CLI. ESLint rules that those files trip are scoped off in `eslint.config.mjs`.
- Compose feature components (in `src/components/[feature]/`) from these primitives.
- Theme tokens and dark mode live in `src/app/globals.css` via the Tailwind v4 `@theme` directive (no JS config).
